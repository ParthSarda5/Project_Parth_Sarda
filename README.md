## 1. Project Overview

This project trains a **ResNet-18 convolutional neural network** to perform binary classification of simulated astronomical survey images: given a 64×64 pixel image of a galaxy field, does it contain a **strong gravitational lens** or not?

## 2. Scientific Background

**What is gravitational lensing?**  
When a massive foreground galaxy sits between Earth and a more distant background galaxy, its gravitational field bends the light from the background source. Depending on the alignment, this produces characteristic arc-like distortions or complete Einstein rings in the observed image.

**Why does automated detection matter?**  
Strong gravitational lenses are rare — roughly one in every few thousand galaxy images in large surveys. Upcoming missions such as the Euclid space telescope and the Rubin Observatory's LSST will image hundreds of millions of galaxies, making manual inspection completely infeasible. CNN-based classifiers can screen candidates at the rate of millions of images per hour.

**Why CNNs are the right tool:**  
- Lensing arcs are spatially localised, curved, and appear at varying orientations — precisely the kind of feature that convolutional filters detect hierarchically.
- Lower layers learn edge and curvature detectors; deeper layers compose these into arc-like structures.
- CNNs trained end-to-end on raw pixel data outperform classical morphology-based classifiers (ellipticity, Sérsic fitting) because they learn discriminative features that are not hand-designed.

**Why simulation-based data?**  
Real confirmed strong lenses are too rare to provide sufficient training data. The simulation approach used here — generating lensed images by ray-tracing via `lenstronomy` — is the standard practice in the astrophysics community (Metcalf et al. 2019; Petrillo et al. 2019; Davies et al. 2019).

## 4. About the Data Format

The data files use **NumPy's `.npy` format**. This is the native raw format of the dataset for the following reasons:

**Why not `.jpg` or `.png`?**  
Each simulated image is a `float32` array whose pixel values span multiple orders of magnitude — a bright galaxy core may be 1000× brighter than a faint lensing arc. JPEG and PNG are 8-bit integer formats that would quantise or clip this dynamic range and destroy the faint signal. The `.npy` format preserves full floating-point precision.

The submission spec states data should be *"in the form found in the raw dataset"* — and the raw form of `lenstronomy` output is a NumPy array.

**How to visually inspect `.npy` images:**

```python
import numpy as np
import matplotlib.pyplot as plt

img = np.load('data/lens_01.npy')          # (64, 64) float32
print(f"Shape: {img.shape}  Min: {img.min():.4f}  Max: {img.max():.4f}")

# Log-stretch compresses the high dynamic range for visualisation
stretched = np.log1p(np.maximum(img, 0))
plt.imshow(stretched, cmap='inferno', origin='lower')
plt.colorbar(label='log(1 + flux)')
plt.title('Gravitational Lens Example')
plt.show()
```

A pre-rendered visual grid of all 20 example images is saved at `data/preview.png` — open it in any image viewer without any code.

## 7. Model Architecture

```
Input: (B, 3, 64, 64)
         ↓
    [ResNet-18 backbone]
    conv1 → bn → relu → maxpool
    layer1 → layer2 → layer3 → layer4
    global average pool → (B, 512)
         ↓
    Linear(512 → 1)
         ↓
Output: scalar logit per image
        sigmoid(logit) → probability of being a lens
```

**Key design choices:**

- **Pretrained backbone:** ImageNet-1K weights initialise the backbone. Even though astronomical images look nothing like natural photographs, the low-level feature detectors (edge, curve, blob detectors) transfer effectively and dramatically speed up convergence.
- **Binary head:** A single `Linear(512 → 1)` replaces the original 1000-class head. `BCEWithLogitsLoss` is used instead of `CrossEntropyLoss` for numerical stability.
- **3-channel input:** The greyscale image is repeated across 3 channels to match the ImageNet-pretrained backbone's expected input shape — a standard technique for single-channel scientific images.

**Preprocessing pipeline** (identical at training time and inference time):

| Step | Operation | Reason |
|------|-----------|--------|
| 1 | `log1p(max(x, 0))` | Compresses astronomical dynamic range |
| 2 | Per-image min-max normalisation → [0, 1] | Stable training signal |
| 3 | Greyscale → 3-channel (repeat) | ImageNet backbone compatibility |
| 4 | Random flip (H+V) + 180° rotation *(train only)* | Lenses are rotationally isotropic |
| 5 | Normalise: mean=0.5, std=0.5 → [−1, 1] | Standard ImageNet normalisation |

**Why full 360° rotation augmentation?**  
Gravitational arcs have no preferred orientation in the sky — the angle at which a lens appears depends entirely on the line-of-sight geometry, which is random. Full rotational augmentation is therefore physically motivated (not just a regularisation trick) and substantially improves generalisation to unseen orientations.

## 8. Training Pipeline

**Optimiser:** Adam with weight decay `1e-4`  
**LR schedule:** Cosine annealing from `1e-4` → `1e-6` over the full run  
**Early stopping:** Patience of 8 epochs (stops if validation AUC does not improve)  
**Loss function:** `BCEWithLogitsLoss` (numerically stable binary cross-entropy)  
**Data split:** 70% train / 15% validation / 15% test (reproducible via fixed seed)

### 9.5 Grad-CAM Interpretability

Grad-CAM (Gradient-weighted Class Activation Mapping) was applied to the final convolutional layer (`backbone.layer4[-1].conv2`) to visualise which spatial regions drive the model's predictions. Results confirm the model has learned physically meaningful features:

- **Lensed images:** Activation concentrates on the arc-like ring structures around the central lens galaxy — the model is detecting the actual lensing signal, not image artefacts.
- **Non-lensed images:** Activation is diffuse, spread across the galaxy core without any ring-like structure.

This is the key qualitative validation — a model that achieves perfect accuracy by memorising noise or dataset artefacts would not produce spatially coherent activations aligned with the arc physics.

To reproduce the Grad-CAM plots, run **Cell 8** of `run_and_verify.ipynb`.

### Re-running training programmatically

```python
import torch, torch.nn as nn
from torch.optim import Adam
from interface import TheModel, the_trainer, the_dataloader, total_epochs

model                       = TheModel(pretrained=True)
train_loader, val_loader, _ = the_dataloader()
optimizer                   = Adam(model.parameters(), lr=1e-4, weight_decay=1e-4)
loss_fn                     = nn.BCEWithLogitsLoss()

# train_model returns a history dict with keys:
# train_loss, train_acc, val_loss, val_acc, val_auc
history = the_trainer(
    model, total_epochs, train_loader, loss_fn, optimizer,
    val_loader=val_loader   # enables early stopping by AUC
)
```

### Running inference on new images

```python
from interface import the_predictor

results = the_predictor([
    'data/lens_01.npy',
    'data/nonlens_01.npy',
    'my_fits_image.fits',
    'my_png_image.png',
])

for r in results:
    print(f"{r['path']}  →  {r['prediction']}  (p={r['probability']:.4f})")
```

Each result is a dict:
```python
{
    'path'       : str,           # input file path
    'prediction' : 'Lens' | 'Non-Lens',
    'probability': float,         # sigmoid output, 0–1
}
```

**Accepted input formats:** `.npy`, `.fits` / `.fit` / `.fts`, `.png`, `.jpg` / `.jpeg`

## 12. Reproducibility

The full pipeline is deterministic given the same seed:

- Data generation: `np.random.default_rng(config.random_seed)` throughout `generate_data.py`
- Train/val/test split: fixed permutation from `np.random.default_rng(config.random_seed)` in `dataset.py`
- The 20 example files in `data/` are fixed (pre-generated, committed to repo)

To fully reproduce the reported results:
```bash
python3 generate_data.py        # generates data/images.npy, data/labels.npy
python3 train.py                # trains for up to 30 epochs, saves checkpoint
python3 predict.py              # runs inference on data/*.npy example files
```
